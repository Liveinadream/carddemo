# CardDemo 项目结构与功能总结

本项目是一个基于 **Godot Engine (C#)** 开发的卡牌类游戏框架，包含了卡牌加载、牌组管理、对话系统、场景切换及基础框架封装。

---

## 1. 项目目录结构概览

```text
carddemo/
├── Assets/                # 静态资源、CSV数据、翻译文件
├── CardImg/               # 卡牌图片资源（包含AI生成图片）
├── Data/                  # 数据持久化模型 (PlayerInfo, DeckSave)
├── Dialogue/              # 对话配置文件 (.dialogue)
├── Scenes/                # Godot 场景文件 (*.tscn)
│   ├── Cards/             # 各类卡牌实例场景
│   ├── Deck/              # 牌组、手牌布局场景
│   ├── Dialogue/          # 对话气泡、UI组件
│   ├── Site/              # 地点/关卡场景
│   └── UI/                # 菜单、启动、信息展示界面
├── Scripts/               # C# 逻辑脚本
│   ├── Baselib/           # 核心底层框架 (Singleton, EventBus, Storage)
│   ├── Cards/             # 卡牌数据加载与逻辑
│   ├── Deck/              # 牌组管理与交互
│   ├── Dialogue/          # 对话逻辑封装
│   ├── Event/             # 全局事件定义
│   ├── General/           # 全局管理器与路径配置
│   ├── Site/              # 场景特定逻辑
│   └── UI/                # 界面控制逻辑
├── addons/                # 第三方插件 (Dialogue Manager)
└── doc/                   # 项目文档与设计思路
```

## 2. 核心模块功能描述

### **2.1 基础框架层 (Baselib)**
这是项目的底层支撑，提供了常用的单例模式和通用工具：
- **Singleton.cs**: 泛型单例模板。
- **EventBus.cs**: 订阅者模式，实现模块间的解耦通信。
- **StorageManager.cs**: 基于 JSON 的本地存档与读取系统。
- **UIManager.cs / ResLoader.cs**: 统一的资源加载与 UI 管理入口。

### **2.2 卡牌系统 (Cards)**
负责卡牌从数据到显示的完整流程：
- **CardDataLoader.cs**: 从 `Assets/cardsInfo.csv` 解析 CSV 数据。
- **CardData.cs**: 定义卡牌的基础属性（名称、类型、价格、权重等）。
- **Card.cs**: 卡牌的基础交互脚本（拖拽、点击、显示）。
- **NpcCard / SiteCard / ShopCard**: 针对不同用途扩展的卡牌子类。

### **2.3 牌组管理 (Deck)**
处理卡牌在空间上的组织方式：
- **Deck.cs**: 通用牌堆逻辑。
- **HandDeck.cs**: 专门用于管理玩家手牌的布局与交互。
- **ShopCardDeck.cs**: 商店页面的卡牌展示与选购逻辑。

### **2.4 对话与交互 (Dialogue)**
利用 `Dialogue Manager` 插件实现动态剧情：
- **DialogueBalloon.cs**: 自定义对话框逻辑。
- **Dialogue/dialogue.dialogue**: 存储剧情和商店对话文本。
- **NPCManager.cs**: 统一管理场景中的 NPC 及其交互。

### **2.5 场景与全局管理 (General)**
- **GlobalManager.cs**: 缓存全局节点（如 UI 层、暂停菜单），提供跨场景访问。
- **GameNormalScene.cs**: 定义场景路径、UID 以及全局通用的常量。
- **PathManager.cs**: 集中管理资源路径。

## 3. 数据与资源流转

1.  **数据源**: 所有卡牌属性配置在 `Assets/cardsInfo.csv` 中。
2.  **加载流**: `CardDataLoader` 读取 CSV -> 转换为 `CardData` 对象 -> 由 `CardManager` 分发给 `Deck`。
3.  **持久化**: 玩家信息和牌组状态通过 `StorageManager` 保存到 `user://save_data/` 下的 JSON 文件。
4.  **多语言**: 翻译文件位于 `Assets/*.translation`，支持本地化显示。
